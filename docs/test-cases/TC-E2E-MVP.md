## EP01 组织与员工主数据

### TC-EP01-001 【P0】组织树加载
- **类型**: 正向  
- **前置**: admin 已登录  
- **步骤**:
  1. 访问 `/org/tree`
  2. 等待树形组件渲染
  3. 展开根节点
- **预期**:
  - 树根显示"演示科技有限公司"
  - 至少有 5 个子部门（总经办/研发/产品/销售/人事）
  - 部门节点可点击展开/折叠
  - 接口 `/org/departments/tree?companyId=1` 返回 200
- **关联 AC**: Epic01 AC-1

### TC-EP01-002 【P0】员工列表分页与搜索
- **类型**: 正向  
- **前置**: admin 登录，跳转 `/hr/employees`  
- **步骤**:
  1. 默认进入列表，查看首屏
  2. 关键字输入"研"，点击搜索
  3. 状态筛选 "ACTIVE"
  4. 翻页至第 2 页
- **预期**:
  - 列表显示 `工号/姓名/部门/状态/入职日期`
  - 关键词过滤生效
  - 状态标签颜色：ACTIVE=success、PROBATION=warning、ON_LEAVE=info、TERMINATED=danger
  - 翻页有 total 计数
  - 敏感字段（身份证/手机号）列表页脱敏
- **关联 AC**: Epic01 AC-10

### TC-EP01-003 【P0】员工详情弹窗
- **类型**: 正向  
- **步骤**:
  1. 列表点击任一员工
  2. 等待弹窗出现
- **预期**:
  - 弹窗显示 `BASIC_FIELDS`（工号/姓名/性别/出生日期/邮箱/部门等）
  - 教育经历 / 工作经历 / 合同作为子表展示
  - admin 可见明文
- **关联 AC**: Epic01 AC-10

### TC-EP01-004 【P1】部门树 CRUD
- **类型**: 正向  
- **步骤**:
  1. 在 `/org/tree` 顶级新增"测试部门A"
  2. 选中"测试部门A"，新增子部门"测试子部门A1"
  3. 编辑"测试子部门A1"为"测试子部门A1-改"
  4. 删除"测试子部门A1-改"
  5. 删除"测试部门A"
- **预期**:
  - 4 个操作均返回 200
  - 树视图实时刷新
  - 删除有空员工的部门允许
- **关联 AC**: Epic01 AC-1

### TC-EP01-005 【P1】删除有员工的部门被拒
- **类型**: 异常  
- **步骤**:
  1. 选择存在员工的部门（如"研发部"）点击删除
- **预期**:
  - 后端返回 4xx，错误码 `DEPT_HAS_ACTIVE_EMPLOYEE`
  - 前端 toast 提示"该部门存在在职员工，请先迁移"
- **关联 AC**: Epic01 AC-7

### TC-EP01-006 【P1】员工状态机 - 入职
- **类型**: 正向  
- **步骤**:
  1. 通过后端 curl `POST /api/hr/employees/hire-form` 创建入职单
  2. 审批通过
  3. 查询员工列表确认状态变更
- **预期**:
  - 员工状态 PENDING_HIRE → ACTIVE
  - 自动创建账号
  - 审计日志有 1 条 hire 记录
- **关联 AC**: Epic01 AC-3

### TC-EP01-007 【P1】员工状态机 - 调岗
- **类型**: 正向  
- **步骤**:
  1. `POST /api/hr/employees/transfer-form` 提交调岗单
  2. 审批
- **预期**:
  - 旧任职记录 end_date = 新生效日 - 1
  - 新任职记录 active = 1
  - 员工只有 1 条 active 任职
- **关联 AC**: Epic01 AC-4

### TC-EP01-008 【P2】身份证号查重
- **类型**: 异常  
- **步骤**:
  1. 重复提交一个已存在身份证的入职单
- **预期**:
  - 后端返回 4xx 错误码 `ID_CARD_DUPLICATE`
  - 提示"已存在 [姓名]，请走重新雇佣流程"
- **关联 AC**: Epic01 AC-2

## EP02 考勤与假期

### TC-EP02-001 【P0】班次管理 - 新建/编辑/删除
- **类型**: 正向  
- **前置**: admin 登录  
- **步骤**:
  1. 访问 `/attendance/leave/new` 旁的"班次排班"页（ShiftSchedule）
  2. 新建班次"朝九晚六"：开始 09:00，结束 18:00
  3. 编辑为"朝九晚六-改"
  4. 删除该班次
- **预期**:
  - 接口 `/attendance/shifts` 增删改均 200
  - 列表实时刷新
- **关联 AC**: Epic02 AC-1

### TC-EP02-002 【P0】员工提交请假
- **类型**: 正向  
- **步骤**:
  1. admin 访问 `/attendance/leave/new`
  2. 假期类型=年假，2026-07-01 ~ 2026-07-02，2 天，备注"休假"
  3. 提交
- **预期**:
  - 跳回 ESS 或 toast "提交成功"
  - 后端生成请假单 status=submitted
  - 余额扣减（如余额 ≥ 2）
- **关联 AC**: Epic02 AC-3、AC-5

### TC-EP02-003 【P1】请假天数自然日计算
- **类型**: 边界  
- **步骤**:
  1. 提交 2026-07-10 ~ 2026-07-12
  2. 提交 0.5 天单（半天）
- **预期**:
  - 跨 2 自然日 → 3 天（首尾含）
  - 0.5 天单可提交
- **关联 AC**: Epic02 AC-2

### TC-EP02-004 【P1】余额不足拒绝
- **类型**: 异常  
- **步骤**:
  1. 假设年假余额 = 1 天
  2. 提交 2 天年假
- **预期**:
  - 返回 4xx 错误码 `LEAVE_BALANCE_INSUFFICIENT`
  - toast 提示"余额不足"
- **关联 AC**: Epic02 AC-3

### TC-EP02-005 【P1】请假日期与已批冲突
- **类型**: 异常  
- **步骤**:
  1. 已有 7/1~7/2 已批请假
  2. 提交 6/30~7/1 区间
- **预期**:
  - 返回 4xx 错误码 `LEAVE_DATE_OVERLAP`
- **关联 AC**: Epic02 AC-4

### TC-EP02-006 【P1】跨月请假整段审批
- **类型**: 边界  
- **步骤**:
  1. 提交 2026-07-30 ~ 2026-08-03 区间
- **预期**:
  - 一次提交 5 天，整段审批
- **关联 AC**: Epic02 AC-6

### TC-EP02-007 【P2】假期年度结算
- **类型**: 正向  
- **步骤**:
  1. admin 触发年度结算（`POST /api/attendance/settlements`）
- **预期**:
  - 年假 > 5 天的部分清零
  - 调休 2 月内有效
  - 事假/病假当年清零
- **关联 AC**: Epic02 AC-9

## EP03 薪酬基础

### TC-EP03-001 【P0】工资周期创建
- **类型**: 正向  
- **步骤**:
  1. admin 访问 `/payroll/runs`
  2. 输入"2026-06"，点击"创建周期"
- **预期**:
  - 周期列表新增 1 条
  - 提示"周期已创建"
- **关联 AC**: Epic03 AC-1

### TC-EP03-002 【P0】工资 run 触发计算
- **类型**: 正向  
- **步骤**:
  1. 选中周期 → "新建批次"
  2. 点击"执行计算"
- **预期**:
  - 返回 200，返回 detailId 数组
  - run 状态 DRAFT → CALCULATED
- **关联 AC**: Epic03 AC-1、AC-2

### TC-EP03-003 【P0】run 锁定后只读
- **类型**: 正向  
- **步骤**:
  1. 对一个 CALCULATED run 点击"锁定"
  2. 尝试通过接口 PUT 修改某 detail 字段
- **预期**:
  - 锁定接口 200
  - 后续 PUT 返回 403
- **关联 AC**: Epic03 AC-3

### TC-EP03-004 【P1】金额 BigDecimal 精度
- **类型**: 边界  
- **步骤**:
  1. 维护薪资档案：基本工资 = 3333.3333
  2. 触发 run
- **预期**:
  - DB 中实际存储为 3333.3333（保留 4 位小数）
  - 不出现 3333.33 截断
- **关联 AC**: Epic03 测试策略 4.3

### TC-EP03-005 【P1】实发为负 - 失败
- **类型**: 异常  
- **步骤**:
  1. 员工薪资档案：基本工资 = 100
  2. 触发 run
- **预期**:
  - 该员工列入异常清单
  - run 状态可能仍为 CALCULATED 但明细标记 fail
- **关联 AC**: Epic03 AC-2

### TC-EP03-006 【P1】红冲生成反向 run
- **类型**: 正向  
- **步骤**:
  1. 对一个 LOCKED run 点击"红冲"
- **预期**:
  - 生成 1 个 REVERSED run，金额为原 run 负向镜像
  - 必须有新的正向 run 才能再次发起
- **关联 AC**: Epic03 AC-4

## EP04 招聘与入职

### TC-EP04-001 【P0】职位列表
- **类型**: 正向  
- **步骤**:
  1. admin 访问 `/recruit/offers`
- **预期**:
  - 列表加载候选人/职位
  - 字段完整
- **关联 AC**: Epic04 AC-1

### TC-EP04-002 【P0】候选人新增
- **类型**: 正向  
- **步骤**:
  1. 点击"新增候选人"，填写基础信息
  2. 提交
- **预期**:
  - 列表新增
  - 接口 `POST /api/recruit/candidates` 返回 200
- **关联 AC**: Epic04 AC-1

### TC-EP04-003 【P0】Offer 创建
- **类型**: 正向  
- **步骤**:
  1. 选中候选人 → "创建 Offer"
  2. 提交薪资、入职日
- **预期**:
  - Offer 状态 DRAFTED
- **关联 AC**: Epic04 AC-5

### TC-EP04-004 【P1】Offer 接受触发草稿员工
- **类型**: 正向  
- **步骤**:
  1. 对 DRAFTED Offer 点击"提交审批" → "接受"
- **预期**:
  - 5 秒内员工档案出现 status=PENDING_HIRE
  - 员工编号已分配
- **关联 AC**: Epic04 AC-4

### TC-EP04-005 【P1】Offer 审批后金额不可改
- **类型**: 异常  
- **步骤**:
  1. 对 APPROVED Offer 尝试调薪
- **预期**:
  - 返回 4xx 错误码 `OFFER_LOCKED`
- **关联 AC**: Epic04 AC-3

### TC-EP04-006 【P1】简历附件类型/大小限制
- **类型**: 异常  
- **步骤**:
  1. 上传 11MB pdf
  2. 上传 exe
- **预期**:
  - 11MB 拒绝，提示"文件超过 10MB"
  - exe 拒绝，提示"格式不允许"
- **关联 AC**: Epic04 AC-8

## EP05 绩效

### TC-EP05-001 【P0】绩效列表
- **类型**: 正向  
- **步骤**:
  1. admin 访问 `/performance/reviews`
- **预期**:
  - 列表加载所有绩效单
  - 显示员工/周期/状态/最终得分
- **关联 AC**: Epic05 AC-1

### TC-EP05-002 【P0】绩效自评提交
- **类型**: 正向  
- **步骤**:
  1. 选中绩效单 → "自评"
  2. 录入评分（在 1-5 范围）
  3. 提交
- **预期**:
  - 绩效单状态 → self_review
- **关联 AC**: Epic05 AC-5

### TC-EP05-003 【P0】上级评分计算最终得分
- **类型**: 正向  
- **步骤**:
  1. 自评 4 分
  2. 上级评 5 分
  3. 提交
- **预期**:
  - 最终得分 = 4×0.3 + 5×0.7 = 4.7
  - 状态 → completed
  - 字段锁定
- **关联 AC**: Epic05 AC-2

### TC-EP05-004 【P1】评分越界
- **类型**: 异常  
- **步骤**:
  1. 自评录入 6 分
- **预期**:
  - 拒绝，提示"评分必须 1-5"
- **关联 AC**: Epic05 AC-5

### TC-EP05-005 【P2】同一员工同一周期唯一
- **类型**: 异常  
- **步骤**:

## EP06 ESS/MSS 自助服务

### TC-EP06-001 【P0】ESS 首页加载
- **类型**: 正向  
- **步骤**:

### TC-EP06-003 【P1】ESS 工资条只读
- **类型**: 正向  
- **步骤**:

- **预期**:

## 横切（登录/权限/审计/异常）

### TC-CROSS-001 【P0】登录成功
- **类型**: 正向  
- **步骤**:
  1. 访问 `/login`
  2. 用户名 admin，密码 Admin@2026
  3. 提交
- **预期**:
  - 接口 `POST /api/auth/login` 返回 200
  - data.accessToken 非空
  - uid=1, username=admin
  - 跳转到 `/dashboard`
- **关联 AC**: 7.1

### TC-CROSS-002 【P1】登录失败 - 错误密码
- **类型**: 异常  
- **步骤**:
  1. 密码输入 "wrong"
- **预期**:
  - HTTP 401，错误码 `AUTH_INVALID_CREDENTIALS`
  - 提示"用户名或密码错误"
- **关联 AC**: 7.1

### TC-CROSS-003 【P1】连续输错 5 次锁定
- **类型**: 异常  
- **步骤**:
  1. 同一账号 5 次错误密码
  2. 第 6 次正确密码
- **预期**:
  - 5 次后提示"账号已锁定 30 分钟"
  - 6 次正确密码仍被拒
- **关联 AC**: 7.1

### TC-CROSS-004 【P2】Swagger UI 可访问性
- **类型**: 正向  
- **步骤**:
  1. 访问 `/swagger-ui/index.html`（仅 dev profile 启用）
- **预期**:
  - 生产 profile 应 404 或被拒绝
  - dev profile 应能渲染 OpenAPI
- **关联 AC**: 安全修复 P1-2
