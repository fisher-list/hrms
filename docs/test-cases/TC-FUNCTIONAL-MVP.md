# HRMS MVP 功能测试用例

> 测试环境：本地 H2 或 Docker PostgreSQL
> 测试账号：admin / Admin@2026（首次登录强制改密）
> 创建日期：2026-06-18

---

## 一、登录与安全

| 编号 | 用例 | 步骤 | 预期结果 |
|------|------|------|---------|
| TC-01 | 正常登录 | 打开 /login，输入 admin / Admin@2026，点登录 | 登录成功，跳转仪表盘 |
| TC-02 | 密码错误 | 输入 admin / wrong，点登录 | 提示"用户名或密码错误" |
| TC-03 | 强制改密 | 首次登录后 | 自动跳转 /change-password |
| TC-04 | 弱密码拒绝 | 新密码输入 123 | 提示"密码至少8位，需包含大小写字母和数字" |
| TC-05 | 含用户名拒绝 | 新密码输入 Admin@2026 | 提示"密码不能包含用户名" |
| TC-06 | 改密成功 | 输入旧密码，新密码 Test@2026，确认一致 | 提示成功，跳转登录页 |
| TC-07 | 限流 | 连续输错密码 6 次 | 第6次返回"请求过于频繁" |
| TC-08 | Token 过期 | 登录后等待 30 分钟不操作 | 下次请求自动刷新 token |

## 二、员工管理

| 编号 | 用例 | 步骤 | 预期结果 |
|------|------|------|---------|
| TC-10 | 查看员工列表 | 点"员工管理"菜单 | 显示员工列表，有分页 |
| TC-11 | 搜索员工 | 在搜索框输入"张" | 筛选出姓名含"张"的员工 |

| TC-13 | 编辑员工 | 点某员工"编辑" → 修改手机号 → 保存 | 手机号已更新 |
| TC-14 | 查看详情 | 点某员工"详情" | 显示基本信息+教育/工作/合同/银行等子表 |
| TC-15 | 离职操作 | 选中员工 → 点"办理离职" | 状态变为"已离职" |

## 八、权限验证

| 编号 | 用例 | 步骤 | 预期结果 |
|------|------|------|---------|

## 测试顺序建议

1. **先测登录**（TC-01 ~ TC-08）— 确认安全功能正常
2. **再测组织**（TC-20 ~ TC-25）— 建好部门/岗位基础数据
3. **然后员工**（TC-10 ~ TC-15）— 建好人员数据
4. **再测业务流**（考勤→薪酬→招聘→绩效）— 有数据才能跑通
5. **最后测权限**（TC-70 ~ TC-72）— 用不同账号验证隔离
